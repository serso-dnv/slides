Automated Testing for Web Applications

A Practical Showcase from Unit to End-to-End

A quick reminder

Unit tests

  • Testing tiny pieces of code, one function at a time
  • Think of unit tests like checking each ingredient before baking a cake
  • Super quick to run
  • Helps catch small problems early
  • Written by developers while building software

End-to-End (E2E) Tests

  • Tests the whole application as a user would use it
  • Like walking through the entire cake-making process from start to finish
  • Checks if everything works together smoothly
  • Simulates real user actions from beginning to end
  • Slower and more complex than unit tests

The web application

Built using a Typescript stack, combining a React-based frontend with a Node.js and Express backend.

Turborepo helps manage the project, making development faster and more efficient by optimizing build processes and code sharing.

Web-client (Frontend)

Interacts with users to collect and display data.

React: VUI, TanStack Query, TanStack Router.

Web-server (Backend)

Node/Express.js. Processes requests, business logic, and database interactions.

Why Automated Testing?

  • Faster feedback during development.
  • Prevents regressions and ensures stability.
  • Improves developer confidence.

Unit Testing

  • Vitest: For frontend components, hooks and utilities.
  • Jest: For backend controllers and utilities.

E2E Testing

  • Microsoft Playwright
  • Validates user journeys, ensuring seamless application functionality.
  • Intercepting API calls
